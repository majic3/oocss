Openning a box of Legos (or how OOCSS helped me stop moaning and enjoy the Casscade)

Prior to OOCSS (which I see as a methodology more than framework) I had a real distain for CSS at times. Since watching the Slideshare (above) I learned to use & reuse css effectively.

  1. Easy debugging (grids,template etc)
  2. Vast range of browser compatiblity (even ie5.5)
  3. Clean & Clear CSS - that can scale as needs of site expand (if practised right); Bloat free
  4. Perfect for use within a CMS (more on this soon)
  5. Pick n Mixin(able) potential (more on this soon)
  6. Easy to instruct to others eg Designers

The only drawback I have found is that you have more markup

Also the project has cause controversy since many Consider its very name to be an Oxymoron or a Contradition in terms:

“CSS can't be OO since CSS is not a programming language”

My view on this is that the rules of a stylesheet within an OOCSS styled site are orientated to the objects used and reused within the site. I view the application of styles much like a Venn diagram and this helps a) prevent bloat whislt b) keeping a consistent style throughout the site

Effectively I feel that Pandoras Box is firmly sealed!
