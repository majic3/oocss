Forms

Design principles

All form elements share the same HTML structure, just like the OOCSS modules do. This applies for all three layout options. So there's no dependency between backend-side form generation and frontend presentation.

Inputs are inside the label. This makes a for attribute for the label and an id for the input redundant for modern browsers. Only IE5/6/7 need them if you want to focus/trigger the input by clicking on the respective label.

There are no fixed widths. A field is always as wide as its container allows. Field styling is done by a surrounding span. The input itself has no border, margin and padding and is set to 100% width. All form elements can be easily nested with the OOCSS grid system.

Checkboxes and Radios are positioned absolutely. This prevents any presentational inconsistency between browsers and makes styling easy.
